Demo Name DataSnap XE multiplatform demos
Setup In order to run the Unit Test, create a DataSnap alias on Data Explorer and call as "Local Server", define the hostname and port 211
Classes demonstrated

TDSServer and related classes
Description DataSnapXE demos illustrate a Delphi DataSnap Server connecting with DataSnap Clients built on four project types: Delphi; C++Builder; Prism (.NET); and RadPHP.
  • Open djsongrp.groupproj to see the following projects:

  • DelphiDSServer project: This server by default runs on TCP/IP port 211 and HTTP port 8010, and also connects with an InterBase database located in the database folder. This server should be run without debugging before any clients are run.

  • DelphiDSClient project: A Delphi native DataSnap client which connects to the DataSnap Server (DelphiDSServer); this client executes many of the server methods available on the Server.

  • FMXDelphiDSClient project: A Delphi/FireMonkey native DataSnap client which connects to the DataSnap Server (DelphiDSServer); this client get a Data from DS Server and bind to a Grid using LiveBindings.

  • DelphiDSClient\Test project: A Unit Test application for the server methods provided by DelphiDSServer; to run this application, you need to define the DataSnap Connection Alias in Data Explorer. By default the application uses "LOCAL SERVER". Some of the unit tests use an XML file passed as a runtime parameter to test dynamic data, which by default is DataSnapTests.xml, found in the Tests directory. You can define the parameter using the IDE Run | Parameters menu.

  • CBuilderDSClient project: A C++Builder DataSnap client which connects to the DataSnap Server (DelphiDSServer.) It also executes many of the server methods available on the Server.

  • RadPHPDSClient: A PHP DataSnap client which connects to the DataSnap Server (DelphiServer.) This server uses a REST connection, by default to the hostname/post of localhost:8010; to change these parameters update the host and protocol properties on ClientModuleProxy.php->DSRestConnection1.
